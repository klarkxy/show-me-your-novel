# 我把同一段提示词喂给了 9 个大模型，让它们各写一部十万字的长篇小说

这个项目我做了差不多两个月，核心只有一句话：**把同一份小说提示词丢给不同的大模型，拿到不同的成品，并排放在网页上让人直接对比**。

## 我为什么要做这件事

去年 11 月开始，国产大模型里能稳定写中长篇的选项变多了——DeepSeek、GLM、Kimi、Qwen、MiniMax、MiMo、米哈游的那个 MiMo…… 我自己一个一个用过，体感是「都不错，但也都不像人」。

然后我有一个很具体的疑问：**「不像人」到底是哪个维度的不像？**

是句子层面的啰嗦？是情节层面的套路化？是人物动机的扁平？还是说，**它就是缺少某些更难描述的东西——节奏、留白、人物自己冒出来的怪癖**？

这些维度我没法靠「让模型写一段开头」回答。我得让它们**各写一整本**，然后从头到尾读。

光我自己读也不够，**得让别人也能读**。而且不能是「看几个片段」那种——得能整本读完，还得方便在两本之间来回翻页。

所以就有了这个项目。

---

## 这个项目里我做了什么

三部小说，每部都用 9 个模型各跑一遍。

题材我挑得很不一样：

- **《赤红二月》**：架空历史 / 政治惊悚。1936 年 2 月 26 日东京，226 事件，但让一个中立派军官做了一件历史里没做过的事。
- **《深圳往事》**：改革开放 / 年代戏。1984 年深圳，何志远从衡阳揣着 80 块钱南下，要在这个刚刚划出来的特区活下去。
- **《陆上孤舟》**：明日方舟二次创作 / 软科幻。罗德岛整艘陆地战舰被空间裂隙甩到了美国东海岸附近的海面上，CNN 正在直播。

每部都是 10 章、每章 2000–3000 字。**统一的提示词**——只把题材、世界观、主角、开篇要求写清楚，**不指定风格、不指定情节走向、不许有元评论**。模型自己写。

读者打开任意一部小说，会先看到「提示词」这一栏——**就是我喂给所有模型的同一份 prompt**。再下面是「各模型作品」一栏，点进去就能选某个模型开始读。

读完一部，可以无缝切到另一部去读同一个章节——**因为是同一份提示词起头，章节顺序、角色名字、世界观都是一致的**，但笔法、节奏、细节完全是模型的自由发挥。

## 几个我自己读下来印象很深的差异

写到这里其实想剧透几段对比，但**请你直接去站点读**——剧透一段 3000 字的开头是毁了这本小说。

我就说几个**结构层面**的发现：

- **谁在用短句谁在用长句**：同一场雪，Kimi 写得像诗，每句都断；GLM 写得像报告，一个复合句塞四个转折。两种读起来完全不一样。
- **人物的心理描写方式**：有的模型直接告诉你「他心里慌了」，有的模型用「他把手从口袋里抽出来，又塞回去」让你自己感觉到慌。
- **节奏**：《赤红二月》里有个 9 个模型都写了的情节转折——某个军官深夜独自去查阅文件——有 3 个模型把这个场景一笔带过，有 5 个模型写满了整整一章。**同样的事件，分配到篇幅的差异居然这么大。**
- **思考过程的可见性**：项目里我加了一个细节，**模型在生成时的思考日志默认是折叠的**。点开能看到它在写这一章之前是怎么想的。DeepSeek 和 Qwen 的思考过程我看着最过瘾——**它们会自己质疑自己上一句的措辞**。这可能是国产模型这两年进步最大的地方。

## 一些我自己的判断（很主观，理性反驳）

不带具体节选的话我大概只能讲**模式**：

1. **短篇选手不一定是长篇选手**。MiMo V2.5 Pro 写单段很漂亮，但写 10 章的时候第 7 章开始人物行为会漂移——它「忘了」主角前面许下的某个承诺。
2. **会写场景不等于会写人物**。Kimi K2.7 的场景描写是我读过最舒服的，但它的男主角常常像摄像机——看到很多事，但很难有「主动去做一件错事」的内驱力。
3. **思考模式开了不一定更好**。开启思考后输出长度变长，但**密度变低**。最后《深圳往事》那部，开了思考的版本普遍比没开的版本多 15%–20% 的字数，但「何志远第一次被骗」那场戏，**没开思考的那个版本反而更狠**。
4. **Anthropic 协议和 OpenAI 协议出来的文本，调性不一样**。这个不是模型本身的问题，是各家训练数据进模型前被格式化处理时留下了味道。

## 这个项目本身长什么样

技术栈很轻：

- 一个 Python 脚本 `runner/generate.py` 直连 LLM API，按章节生成、单章校验、不通过会重试。
- 一个 `scripts/generate_site.py` 把所有模型产出渲染成静态站点。
- 仓库就放在 GitHub 上：[https://github.com/klarkxy/show-me-your-novel](https://github.com/klarkxy/show-me-your-novel)
- 生成的 `docs/` 直接被 GitHub Actions 部署到 GitHub Pages。

API key 只在我本地用，**仓库里没有任何密钥**。

如果你想自己跑一遍换题材，加一个 `novels/<slug>/prompt.md` 写好提示词就行。模型列表在 `config.yaml`，增减都改这一个文件。

## 我接下来想做的

- **多语言版本**。日语和英语的提示词同样试一遍，看国产模型在非母语长篇上的实际能力。
- **加「人工评分」维度**。光靠读者体感不够，我想加一个三人盲评——同章节去作者化，三个读者打分。
- **模型迭代追踪**。Qwen 3.7 → 3.8 这种小版本变化，对长篇写作到底有没有可见影响？每出一个新版本我都重跑一遍《赤红二月》前三章。

---

## 写在最后

大模型写小说这件事，**当下更像 2018 年左右的 AI 绘画**——能用，能用得不错，但离「能让人忘了我在看 AI 写」还有距离。

这个项目的目标不是证明哪个模型最好（这种排名没意义），**是想把「不一样」这件事以最低门槛展示出来**。你读 10 分钟就会发现，**「哪个模型」其实是个错误的提问方式**——同一个模型在不同题材下表现差异巨大；同一个题材下不同模型写出来的东西**根本不像同一种动物**。

我把这堆动物都关在同一个笼子里放出来了。你自己去挑。
